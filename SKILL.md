# 供应链智能表单生成

自动化核查供应链数据明细表并按模板规范生成请验单、入库单、出库单，支持数据一致性核验。

## 核心能力

### 1. 数据明细表核查（11项规则）
使用 `python main.py validate <数据文件路径>` 执行核查：

1. **数据完整性**：必填字段（物料编码、物料名称、规格、单位、品牌、供应商）不能为空
2. **负数检查**：表格中不应出现负数
3. **日期顺序**：采购申请日期 < 请验日期 < 入库日期 < 出库日期
4. **数字精度**：数字类型数据应保留两位小数
5. **出库逻辑**：结存量递减符合事实逻辑（入库数量 - 领料量 = 结存量）
6. **有效期检查**：入库日期必须比有效期早至少6个月
7. **仓库类型**：只能是原材料仓、包材库、标签库之一，不能为空
8. **入库数量计算**：入库数量 = 采购数量 - 请验消耗量

### 2. 生成请验单
使用 `python main.py generate-inspection <数据文件路径> [--output 输出目录]`

每个物料生成一个请验单，包含上下两联（第一联留存，第二联QC），自动填充：
- 样品名称、样品分类、批号、请验部门
- 请验人、请验日期、规格、数量
- 抽样人、抽样数量、物料编码、备注

### 3. 生成入库单
使用 `python main.py generate-inbound <数据文件路径> [--output 输出目录]`

按入库单号、入库日期、仓库类型分组合并，相同条件的物料合并到一张入库单。

### 4. 生成出库单
使用 `python main.py generate-outbound <数据文件路径> [--output 输出目录]`

按出库单号、出库日期、仓库类型分组，支持每个物料多次出库（最多3个批次）。

### 5. 表单核验
使用 `python main.py verify <数据文件路径> [--output 表单目录]`

核验已生成的表单与数据明细表的一致性，检查物料编码、数量等是否对应。

## 工作流程

当用户提供数据明细表时，按以下流程执行：

1. **数据核查**：先执行 validate，确认数据无明显错误
2. **生成表单**：根据需求生成请验单/入库单/出库单
3. **结果核验**：使用 verify 检查生成结果与源数据一致性

如有错误，向用户说明问题所在，建议修正数据后再生成。

## 数据明细表格式要求

- 表头在第4行（header_row=3）
- 关键列位置：
  - B列(1): 物料编码
  - C列(2): 物料名称
  - D列(3): 规格
  - E列(4): 单位
  - F列(5): 品牌
  - G列(6): 供应商
  - H列(7): 样品分类
  - I列(8): 采购数量
  - N列(13): 抽样数量
  - P列(15): 请验日期
  - R列(17): 入库日期
  - T列(19): 入库批号
  - V列(21): 有效期
  - W列(22): 入库仓库
  - X列(23): 入库单号
  - Z列(25): 出库单号1
  - AD列(29): 出库单号2
  - AH列(33): 出库单号3

## 注意事项

- 数据明细表通常为Excel格式（.xlsx）
- 生成的表单严格按照模板格式，保留原有样式
- 入库单和出库单支持多物料合并，请验单每物料独立
- 所有日期格式统一为 YYYY.MM.DD（请验单）或 YYYY-MM-DD（入库/出库单）
